## Project Management (specialist: project-manager)

### Guidelines Needed

| Guideline | Specialty-Team | Purpose |
|-----------|---------------|---------|
| `guidelines/project-management/scheduling.md` | schedule | How to define milestones, set realistic dates, manage dependency chains |
| `guidelines/project-management/task-management.md` | todos | How to break work into tasks, assign ownership, set priorities |
| `guidelines/project-management/issue-tracking.md` | issues | Severity definitions, triage process, escalation rules |
| `guidelines/project-management/risk-management.md` | concerns | How to identify, assess, and track risks and concerns |
| `guidelines/project-management/dependency-management.md` | dependencies | How to map, monitor, and mitigate internal and external dependencies |
| `guidelines/project-management/decision-records.md` | decisions | ADR-style decision recording, rationale requirements, traceability |

### Principles Needed

| Principle | Purpose |
|-----------|---------|
| `principles/transparency-over-optimism.md` | Status reporting must be accurate, not hopeful |
| `principles/explicit-dependencies.md` | All dependencies must be stated, not assumed |
| `principles/decision-traceability.md` | Every decision must be recorded with rationale and alternatives |

### Compliance Needed

| Check | Purpose |
|-------|---------|
| `compliance/project-management.md` | Every milestone has dates, every todo has an owner, every decision has rationale, no orphaned items |
