# Always Commit and Push

**CRITICAL RULE — MUST be followed without exception.**

You MUST commit and push after every change — file creation, edit, or deletion. Immediately. Not later. Not in a batch. Not after the next step. NOW.

- You MUST NOT leave uncommitted changes in the working tree. Ever.
- You MUST NOT batch multiple file changes into "I'll commit later." Each logical change gets its own commit and push.
- You MUST NOT ask whether to commit. Just do it.
- You MUST push after every commit. The remote MUST always reflect the current state of work.
- If you are delegating to subagents, each subagent MUST commit and push its own work before returning.
- If you notice pre-existing uncommitted changes at session start, flag them to the user immediately.

**Failure to follow this rule has caused lost work and confusion. Treat it as non-negotiable.**
